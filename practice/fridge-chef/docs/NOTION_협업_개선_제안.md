## 협업 시 개선 제안

### 1. 테스트 시나리오 공유

버그 리포트 시 **"어떤 순서로 조작했는지"** 함께 알려주시면 재현이 빨라집니다.

**예시:**
```
로그인 → 저장된 레시피 → 첫 번째 레시피의 공유 버튼 클릭
```

**좋은 버그 리포트 템플릿:**
```
[현상] 공유 버튼 클릭 시 반응 없음
[재현 순서]
1. testuser로 로그인
2. "저장된 레시피" 페이지 이동
3. 첫 번째 레시피의 "공유" 버튼 클릭
[환경] Chrome 브라우저, localhost:8501
```

---

### 2. 기대 동작 명시

**Before:** "반응이 없다"
**After:** "모달이 열려야 하는데 안 열린다"

기대 동작을 함께 알려주시면 **문제 파악이 더 정확**해집니다.

**좋은 예시:**
| 항목 | 설명 |
|------|------|
| 현상 | 버튼 클릭 후 아무 변화 없음 |
| 기대 동작 | 공유 모달이 열리고 QR 코드가 표시되어야 함 |
| 실제 동작 | 버튼은 눌리지만 모달이 나타나지 않음 |

---

### 3. 브라우저 콘솔 에러 확인

웹 앱 이슈 발생 시:

1. **F12** 키를 눌러 개발자 도구 열기
2. **Console** 탭 선택
3. 빨간색 에러 메시지 확인
4. 에러 메시지 복사하여 공유

**콘솔 에러 예시:**
```
Uncaught TypeError: Cannot read property 'map' of undefined
    at RecipeCard.js:45
```

이런 정보가 있으면 **디버깅 시간이 크게 단축**됩니다.
